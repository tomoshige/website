    第4章 データのインポートとTidy Data

    スライド こちら

    はじめに

    これまでの章では、プログラミングの基礎としてデータフレーム（行が観測値、列が変数を表す表形式のデータ構造）について学び、既存のデータセットを用いてデータの探索や可視化、変換を行ってきました。

    本章では、データ分析の効率と再現性を高める上で非常に重要な概念である「Tidy Data（整然データ）」について深く掘り下げます。この「Tidy」という言葉は、単に「整理整頓されている」という意味だけでなく、データサイエンティストがデータを扱う上での標準的な形式を指します。

    これまでの演習で用いたデータは、実はすでにTidyな形式になっていました。しかし、実世界のデータが常にそのような都合の良い形で提供されるわけではありません。本章では、さまざまな形式のデータをPythonにインポートする方法を学び、なぜTidy Dataが分析ツールにとって不可欠なのか、そしてどのようにデータをTidyな形式に整形するのかを実践的に習得します。この知識は、今後の統計的モデリングや機械学習の学習においても基盤となります。

    まずは、外部のデータをPython環境に読み込む方法から見ていきましょう。

    1. データインポート

    分析プロジェクトの多くは、外部ファイルに保存されたデータ（自身のコンピュータ上、あるいはウェブ上）を読み込むことから始まります。スプレッドシート形式のデータは、主に以下の形式で保存されています。

    1. カンマ区切り値 (CSV: Comma Separated Values) .csv ファイル:

      • 各行が1つの観測値を表し、行内の各値（フィールド）がカンマで区切られるテキストファイル。
      • 最初の行は、多くの場合、列名（変数名）を示すヘッダー行となる。
      • 最も広く使われているデータ交換フォーマットの一つ。

    2. Excel .xlsx ファイル:

      • Microsoft Excelの標準ファイル形式。
      • データだけでなく、書式設定や数式、グラフなどのメタデータも含むことができる。

    Pythonでこれらのデータを扱う際には、主にpandasライブラリの関数を使用します。

    1.1. 必要なライブラリのインポート

    まず、データ操作や可視化に必要なライブラリをインポートします。

    import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

# seabornのグラフスタイルを設定
sns.set_theme(style="whitegrid")

    1.2. CSVファイルのインポート

    ここではseabornに付属しているirisデータセットを一度CSVファイルとしてColab環境の一時領域に保存し、それをpandasで読み込むという手順でインポートを実演します。

    # seabornからirisデータセットをロード
iris_data = sns.load_dataset('iris')

# irisデータセットをCSVファイルとして保存（Colab環境の一時領域に保存）
csv_file_path = 'iris_dataset.csv'
iris_data.to_csv(csv_file_path, index=False) # index=FalseでDataFrameのインデックスをCSVに書き出さない

# 保存したCSVファイルをpandasで読み込む
iris_from_csv = pd.read_csv(csv_file_path)

# 読み込んだデータの最初の5行を表示
print("CSVから読み込んだirisデータ:")
print(iris_from_csv.head())

    解説: pd.read_csv()関数は、CSVファイルを読み込み、データフレームとして返します。上記のようにローカルファイルパスを指定するほか、URLを直接指定してウェブ上のCSVファイルを読み込むことも可能です。

    1.3. Excelファイルのインポート

    Excelファイルをインポートする場合は、pd.read_excel()関数を使用します。同様にirisデータセットをExcelファイルとして保存し、読み込んでみましょう。

    # irisデータセットをExcelファイルとして保存（Colab環境の一時領域に保存）
excel_file_path = 'iris_dataset.xlsx'
# Excelファイルに書き出すには、'openpyxl'エンジンが必要になることがあります。
# Colabには通常プリインストールされています。
iris_data.to_excel(excel_file_path, index=False, engine='openpyxl')

# 保存したExcelファイルをpandasで読み込む
iris_from_excel = pd.read_excel(excel_file_path)

# 読み込んだデータの最初の5行を表示
print("\nExcelから読み込んだirisデータ:")
print(iris_from_excel.head())
    注意: pd.read_excel() を使用するには、openpyxl や xlrd (古い.xls形式の場合) といった追加のライブラリが必要です。Colab環境では openpyxl が通常プリインストールされています。

    2. Tidy Data（整然データ）

    データ分析を行う上で、データの「形式」は非常に重要です。Hadley Wickhamによって提唱された「Tidy Data」という概念は、データを分析しやすい標準的な構造に整形するための指針を与えてくれます。

    Tidy Dataの3原則:

    1. 各 変数 (variable) が 列 (column) を形成する。
    2. 各 観測値 (observation) が 行 (row) を形成する。
    3. 各種類の 観測ユニット (observational unit) が テーブル (table) を形成する。

    この原則に従ったデータは、pandasでの操作やseaborn、matplotlibでの可視化が格段に行いやすくなります。

    2.1. Tidy Dataの具体例

    多くのデータセットは最初からTidyな形式になっているわけではありません。よく見られるのが「ワイド形式 (wide format)」のデータです。これは、1つの観測に対して複数の変数が横に広がっている形式です。一方、Tidy Dataは「ロング形式 (long format)」または「ナロー形式 (narrow format)」と呼ばれることが多いです。

    例として、seaborn に含まれる mpg (miles per gallon) データセットを使用します。このデータセットには、さまざまな車種の燃費や性能に関する情報が含まれています。

    # seabornからmpgデータセットをロード
mpg_raw = sns.load_dataset('mpg')
print("元のmpgデータ（一部）:")
print(mpg_raw.head())

# origin列を文字列にマッピング (例示のため、元が文字列でも再マッピング)
origin_map = {1: 'usa', 2: 'europe', 3: 'japan'} # 元のデータセットのoriginは数値か文字列
if mpg_raw['origin'].dtype != 'object': # もし数値なら文字列に変換
    mpg_raw['origin_name'] = mpg_raw['origin'].map(origin_map)
else: # 元々文字列 (例: 'usa') ならそのまま使う
    mpg_raw['origin_name'] = mpg_raw['origin']


# デモンストレーションのため、いくつかの車種と特定の性能指標を選び、
# ワイド形式に近いデータを作成します。
# ここでは、各'origin_name'の平均的な性能値を持つワイド形式のデータを作成してみましょう。
mpg_summary_wide = mpg_raw.groupby('origin_name')[['horsepower', 'weight', 'acceleration']].mean().reset_index()

print("\n加工後のmpg要約データ (ワイド形式):")
print(mpg_summary_wide)

    この mpg_summary_wide データフレームは、origin_name ごとに平均の horsepower, weight, acceleration がそれぞれ別の列になっています。これはワイド形式の一例です。

    もし、生産国別・性能指標別の値を比較するグループ化された棒グラフを作成したい場合を考えてみましょう。このグラフでは、以下のような要素が必要です。

    1. X軸: origin_name（生産国名）
    2. Y軸: value（性能指標の値）
    3. 色分け: metric_type（性能指標の種類: horsepower, weight, acceleration）

    現在の mpg_summary_wide の形式では、horsepower, weight, acceleration が独立した列になっているため、単一の metric_type 変数として扱い、グラフの色分けに直接マッピングすることが困難です。

    これを解決するのが、データをTidyなロング形式に変換することです。

    2.2. ワイド形式からTidy（ロング）形式への変換: melt()

    pandasのmelt()関数は、データフレームをワイド形式からロング形式に変換するのに役立ちます。

    mpg_summary_tidy = mpg_summary_wide.melt(
    id_vars=['origin_name'],                         # 変化させずにそのまま保持する列（識別子）
    value_vars=['horsepower', 'weight', 'acceleration'], # ロング形式に変換したい値が含まれる列
    var_name='metric_type',                          # value_varsの列名が格納される新しい列の名前
    value_name='value'                               # value_varsの値が格納される新しい列の名前
)

print("\nTidy形式に変換されたmpg要約データ:")
print(mpg_summary_tidy.sort_values(by=['origin_name', 'metric_type']))

    melt()関数の引数:

    • id_vars: 変換後もそのまま識別子として残す列のリスト。この例では origin_name です。
    • value_vars: 「溶かして」ロング形式にする列のリスト。この例では horsepower, weight, acceleration です。省略した場合、id_vars以外の全ての列が対象となります。
    • var_name: value_varsで指定された列名自体が値として入る新しい列の名前。ここでは metric_type としました。
    • value_name: value_varsで指定された列の実際の数値データが入る新しい列の名前。ここでは value としました。

    変換後の mpg_summary_tidy データフレームはTidy Dataの原則に従っています。 1. origin_name, metric_type, value という各変数が列を形成。 2. 各行が一つの生産国における一種類の性能指標の平均値という観測値を形成。

    このTidyな形式のデータを使えば、目的の可視化が容易になります。

    plt.figure(figsize=(10, 6))
sns.barplot(x='origin_name', y='value', hue='metric_type', data=mpg_summary_tidy)
plt.title('生産国別・性能指標別 平均値')
plt.xlabel('生産国')
plt.ylabel('平均値')
plt.legend(title='性能指標')
plt.show()

    このように、データをTidyな形式にすることで、seabornのようなライブラリが期待する入力形式に適合し、複雑なグラフも簡潔なコードで描画できるようになります。

    3. ケーススタディ: 航空会社の乗客数とTidy Data

    もう一つの例として、seabornに付属している flights データセットを使って、時系列データの整形と可視化を行います。このデータセットは元々Tidyなロング形式（year, month, passengersの3列）で提供されており、meltの直接的な適用対象というよりは、Tidyなデータが如何に分析に適しているかを示す良い例です。 ここでは、Tidyなデータを一度ワイド形式に変換し、それをmelt()を使ってTidyなロング形式に戻す練習をしてみましょう。

    flights_data = sns.load_dataset('flights')
print("元のflightsデータ (ロング形式):")
print(flights_data.head())

    3.1. ロング形式からワイド形式へ: pivot_table()

    まず、このデータを「年を行、月を列、値に乗客数」というワイド形式に変換してみます。これには pivot_table() 関数が使えます。

    flights_wide = flights_data.pivot_table(
    index='year',      # 新しいデータフレームのインデックスとなる列
    columns='month',   # 新しいデータフレームの列となる列
    values='passengers' # 新しいデータフレームの値となる列
)

print("\nワイド形式に変換されたflightsデータ (年 x 月):")
print(flights_wide.head())
    この flights_wide データフレームは、各行が年を表し、各列が月を表し、その交点に乗客数が入っています。これはTidy Dataの原則（特に「各変数が列を形成する」）には従っていません。例えば、「Janの乗客数」「Febの乗客数」...「Decの乗客数」がそれぞれ独立した列名として扱われています。

    3.2. ワイド形式からTidy（ロング）形式へ戻す: melt()

    このワイド形式の flights_wide データを、再びTidyなロング形式（year, month, passengers の3列を持つ形式）に戻してみましょう。melt() を使いますが、flights_wide のインデックス（year）を列に戻してから処理します。

    # インデックス 'year' を通常の列に戻す
flights_wide_reset = flights_wide.reset_index()
print("\nインデックスをリセットしたワイド形式データ:")
print(flights_wide_reset.head())

# meltを使ってTidy (ロング) 形式に変換
# value_vars を指定しない場合、id_vars 以外のすべての列が対象になります
flights_tidy_again = flights_wide_reset.melt(
    id_vars=['year'],              # 識別子として保持する列
    var_name='month',              # 月の列名が格納される新しい列の名前
    value_name='passengers'        # 乗客数が格納される新しい列の名前
)

# 月の順序を元のデータセットのカテゴリ順に合わせる (オプション)
# flights_data['month'] は元々 CategoricalDtype なので、その順序を利用
month_order = flights_data['month'].cat.categories
flights_tidy_again['month'] = pd.Categorical(flights_tidy_again['month'], categories=month_order, ordered=True)


# 整形後のデータを確認 (年と月でソート)
print("\n再びTidy形式に変換されたflightsデータ:")
print(flights_tidy_again.sort_values(by=['year', 'month']).head())

    3.3. Tidyデータを用いた時系列プロット

    Tidy形式になった flights_tidy_again データ（これは元の flights_data と本質的に同じ構造です）を使えば、例えば年ごとの乗客数の推移を月別に色分けして表示するような時系列プロットも容易に作成できます。

    plt.figure(figsize=(12, 7))
sns.lineplot(x='year', y='passengers', hue='month', data=flights_tidy_again, palette='viridis')
# もし元のflights_dataを直接使っても同じグラフが得られます
# sns.lineplot(x='year', y='passengers', hue='month', data=flights_data, palette='viridis')
plt.title('月別・年別 航空会社乗客数の推移')
plt.xlabel('年')
plt.ylabel('乗客数')
plt.legend(title='月', bbox_to_anchor=(1.05, 1), loc='upper left')
plt.tight_layout() # 凡例がグラフエリア外に出る場合などに調整
plt.show()

    このケーススタディでは、pivot_table()で意図的に非Tidyなワイド形式データを作成し、それをmelt()でTidyなロング形式に戻すプロセスを体験しました。実務では、最初からワイド形式で提供されるデータを扱うことが多いため、melt()を使いこなすことは非常に重要です。

    4. まとめ

    本章では、以下の重要な概念とスキルを学びました。

    1. データインポート: pandasを用いてCSVファイル (pd.read_csv()) やExcelファイル (pd.read_excel()) をPythonに読み込む基本的な方法。デモではseabornのirisデータセットを一時ファイルとして利用。
    2. Tidy Dataの概念:
      • Hadley Wickhamによる3つの原則（各変数が列、各観測値が行、各観測ユニットがテーブル）。
      • Tidy Data（主にロング形式）がデータ分析、特にpandasやseabornを用いた処理や可視化においてなぜ重要なのか。
    3. データ整形: pandasのmelt()関数を用いて、非Tidyな「ワイド形式」のデータをTidyな「ロング形式」に変換する方法。
      • id_vars, value_vars, var_name, value_name といった主要な引数の理解。
      • seabornのmpgデータセットを例に、ワイド形式のデータを作成しTidy形式へ変換するデモを実施。
    4. 実践的応用: Tidy Dataを用いることで、複雑な構造のグラフ（グループ化された棒グラフや複数の時系列線グラフなど）を効率的に作成できること。seabornのflightsデータセットを用いて例示。

    データがTidyな形式であることは、この後の講義で学ぶ統計的モデリングや機械学習のタスクにおいても、データの前処理段階で不可欠なステップとなります。

    次章からは、これまでに習得したデータラングリング（データの操作・整形）や可視化のスキルを活かし、データ内の変数間の関係性を明らかにするための「モデリング」の世界へと進んでいきます。
